# 今後の課題・調整メモ

最終更新: 2026-07-11

## 優先度高め

### 会話UIの整理

今の吹き出しは、回答本文、履歴、ソースURL、入力欄、マイク、送信が同居して密集しやすい。

2026-07-11時点で、セリフ表示中のホバーでは本文とソースを残したまま入力欄だけ追加するように改善済み。コピーボタンも文字ラベルからアイコンへ変更済み。とはいえ、情報量が多い時の触りやすさはまだ改善余地がある。

検討案:

- 通常時は本文と最小操作だけ表示し、詳細操作は「…」で展開する。
- コピーはアイコン化済み。次は「本文コピー」「URLコピー」を分けるか、長押し/メニューに逃がすかを検討する。
- ソースURLは本文下に常時表示せず、情報共有系だけ広めに表示する。
- 入力欄を吹き出し内ではなく、キャラ横または下側の専用パネルにする。
- マイクボタンは録音中・文字起こし中・失敗時の状態をもっと分かりやすくする。

### 音声入力の精度改善

現状:

- `ggml-base.bin` は軽いが、聞き間違いが目立つ。
- 短い発話では特に文脈補正が効きにくい。

候補:

- `small` モデルで速度・精度を比較。
- 無音区間カット、音量正規化、短すぎる録音の警告を入れる。
- 文字起こし後に会話文脈で軽く整形する。ただし勝手に意味を変えない。
- 文字起こし結果をすぐ送信せず、必ず入力欄で確認できる状態を維持する。

### 会話の自然さ（次の開発予定・公開作業が済んでから／v0.2候補）

GPT-Live（OpenAIが2026-07-08に出したfull-duplex音声モデル。相づち・割り込み対応、ターン制廃止）級の自然さを、**びくたんの声＝猫使ビィを守ったまま**目指す。実装順:

1. **応答テンポの改善（効果最大）**: 今は「全文生成→全文合成→再生」の直列。ストリーミング応答＋文単位のVOICEVOX合成で、最初の一文から喋り始める（体感5〜15秒→2〜3秒）。Claude APIはストリーミング対応、Claude Code CLIにもストリーム出力モードあり。
2. **つなぎの相づち**: 生成待ちの間に「んー、ちょっと待ってくださいね」等のローカル相づちを即時再生。無言の間をなくす。実装は軽い。
3. **ハンズフリー会話モード**: マイク常時オン＋無音検出（VAD）で自動聞き取り。びくたんの発話中にユーザーが話し始めたら読み上げを止める「割り込み」もここで。whisperローカルなのでプライバシーは維持。
4. **GPT-Live API連携（API公開待ち）**: 2026-07-12時点でAPIは「coming soon」で時期未発表。出たら音声出力は使わず「低遅延の耳と頭」としてだけ使い、口はVOICEVOXを維持する構成を検討。

- ライセンス表記の同梱確認。

## 優先度中

### Windows対応

検討が必要な置き換え:

| macOS現状 | Windows候補 |
|---|---|
| LaunchAgent | スタートアップ登録 / 自動起動設定 |
| メニューバーTray | Electron Tray |
| `afplay` | Electron audio / PowerShell / node audio |
| VOICEVOX.app | VOICEVOX Engine 起動検出、または同梱しない利用 |
| `native/now-playing` | Windows Media Control / PowerShell / 代替なし |
| `pmset` | 代替なし、または音楽反応はアプリ検出に限定 |

### びくたんの個性

やりたいこと:

- 問答で覚えた好みを、自動セリフに自然に混ぜる。
- 「勉強熱心」な感じを増やす。
- メタ発言を減らす。
- 用意されたセリフ感を減らし、会話の地続き感を出す。
- 同じセリフ・似た構文の繰り返しを減らす。

注意:

- 見えないものを見たふりしない。
- ユーザーの状態を決めつけない。
- ポエム・格言・過剰な自己紹介に戻さない。

### 日記・記憶

検討案:

- 日記を「セーブポイント」として扱う。
- 次の日以降に、前日の要点を自然に思い出す。
- 長期記憶は増えすぎるとプロンプトが重くなるため、要約・優先度・期限を持たせる。
- 個人情報やセンシティブな内容は勝手に保存しない。
