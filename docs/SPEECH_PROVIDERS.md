# 音声プロバイダ

びくたんの読み上げ音声は、会話処理から音声エンジン固有の実装を分離している。

## 現在の構成

- `voicevox`: 既定音声。VOICEVOX「猫使ビィ」で、文単位の先行合成に対応する。
- `macos`: VOICEVOXが利用できない場合の代替音声。
- 音量、ミュート、音声再生、口パク、停止、聞き直しはプロバイダの外側にあり、共通で動く。
- `speech-provider-utils.js` が登録情報、能力、フォールバック順を管理する。

各プロバイダは、ID、表示名、音声ファイルを作る `synthesize`、文単位の先行合成へ対応するかを示す能力情報を持つ。

## 本人音声を追加する時

本人音声は既定音声を置き換えず、明示的に選べる追加プロバイダとして接続する。録音、学習データ、生成モデル、音声キャッシュはGitリポジトリと配布物へ含めない。

COEIROINKなどを接続する前に、次を決める。

- ローカル処理だけで完結するか、外部送信があるか。
- 録音・モデルの保存場所と一括削除方法。
- 短文と長文の生成速度、文単位の先行合成への対応。
- 既定のVOICEVOXへ確実に戻す方法。
- 将来のキャラクター差し替え時に、声をアプリ共通・キャラクター別のどちらで選ぶか。
