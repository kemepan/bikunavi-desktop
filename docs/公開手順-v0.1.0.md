# 公開手順（v0.1.0）

準備済みのものを public にするときの手順。ターミナルで上から順に実行するだけでOK。
（GitHub の Web 画面からでも同じことができる。その場合は Settings → Danger Zone → Change visibility → Public、Releases → Draft a new release）

## 0. 事前確認（済んでいるもの）

- ✅ 現在の追跡ファイルに実キー・個人メール・個人用データなし。著者はGitHubのnoreplyアドレス
- ✅ 過去のgit履歴から旧開発環境のローカルパスとnote下書きを除去済み
- ✅ 素材権利の棚卸し（docs/RIGHTS_CHECK.md）
- ✅ ライセンス表記の同梱（THIRD_PARTY_NOTICES.md、.app内にも入る）
- ✅ README に利用条件を明記（コード・素材とも権利保持、個人利用無料）
- ✅ 配布zip: `dist/bikutan-mac-universal-v0.1.0.zip`（361,136,661 bytes、展開・署名・arm64/x86_64検証済み）
- ✅ SHA-256: `d024807958da4aca52149db34219fceb34f00c2be55e56c38934c406314162bb`

## 1. 未コミット分をコミットして push

```bash
cd /path/to/bikunavi-desktop
git add -A
git status   # 内容を確認してから
git commit -m "公開準備: 利用条件・配布資料を整備"
git push
```

## 2. リポジトリを public 化

```bash
gh repo edit kemepan/bikunavi-desktop --visibility public --accept-visibility-change-consequences
```

## 3. Release v0.1.0 を作成して zip を添付

```bash
cd /path/to/bikunavi-desktop
gh release create v0.1.0 "dist/bikutan-mac-universal-v0.1.0.zip" \
  --title "びくたん Desktop v0.1.0" \
  --notes-file docs/リリースノート-v0.1.0.md
```

## 4. 記事の仕上げ（2本構成）

- note下書きはGit管理外の `docs/private/note/` に保存している
- 先に **②導入と使い方編**を仕上げて公開し、URLを確定する
- 次に **①導入秘話編**の〔②のURL〕を埋めて公開する。拡散のメインは①
- 両記事の〔〕を実URLに差し替え
  - Releases URL: `https://github.com/kemepan/bikunavi-desktop/releases/latest`
  - リポジトリ URL: `https://github.com/kemepan/bikunavi-desktop`
  - お茶代（BOOTHブースト0円商品 or noteサポート）と SNS アカウント
  - ①の制作ツール表の〔ペイントソフト名〕〔パーツ分けツール〕〔開発AI〕と制作小話
- スクショを撮って挿入（各下書き内の【スクショ:】位置）

## 取り消したくなったら

- private に戻す: `gh repo edit kemepan/bikunavi-desktop --visibility private --accept-visibility-change-consequences`
- Release 削除: `gh release delete v0.1.0 --cleanup-tag`
- ※ 一度でも public になった内容はクローン・キャッシュされている可能性がある前提で考えること
